### Анализ для авиационного контекста

**Ключевые факторы усталости пилота:**

1. **Decisions_Made (29.09%)** - Количество принятых решений
   - Пилот принимает 100+ решений за рейс
   - Каждое решение увеличивает когнитивную нагрузку

2. **Hours_Awake (28.62%)** - Часы бодрствования
   - FAA ограничивает до 14 часов
   - Корреляция с Time-On-Task усталостью

3. **Error_Rate (18.28%)** - Частота ошибок
   - Критический показатель безопасности
   - Связан с микроснами и вниманием

4. **Task_Switches (10.78%)** - Переключения задач
   - Отвлечения в кабине
   - Переход между частотами, задачами

### Почему выбрана RandomForest?

**Причины выбора (несмотря на меньшую точность LogisticRegression):**

1. **Совместимость sklearn 1.8+**
   - LogisticRegression имеет проблемы с атрибутом `multi_class`
   - RandomForest работает стабильно

2. **Устойчивость к выбросам**
   - Пилоты могут вводить экстремальные значения
   - Модель не даёт сбой на аномалиях

3. **Интерпретируемость для авиации**
   - Feature importance легко объяснить инструкторам
   - Прозрачность для аудита безопасности

4. **Точность 97.82% достаточна**
   - Для safety-critical систем важнее стабильность
   - Ложноотрицательные (пропуск усталости) критичнее ложноположительных

### Детальная математическая модель усталости

#### Архитектура Random Forest

| Параметр | Значение |
|----------|----------|
| Алгоритм | `RandomForestClassifier` (scikit-learn) |
| Количество деревьев | **200** |
| Максимальная глубина | **15** |
| Min samples split | 5 |
| Min samples leaf | 1 |
| Критерий | Gini impurity |
| Признаков на split | `sqrt` (3 из 9) |
| Bootstrap | Да |
| Random state | 42 |

**Математическая формулировка предсказания:**

```
  x_raw = [Hours_Awake, Decisions_Made, ..., Cognitive_Load_Score]   (9 признаков)

  x_scaled = (x_raw - μ) / σ                                          (StandardScaler)

  p(c | x) = (1 / 200) * Σ(t=1..200) I( Tᵗ(x_scaled) = c )          (усреднение голосов)

  ŷ = argmax_c p(c | x)                                              (финальный класс)
```

#### Точные пороговые условия для каждого класса усталости

Модель не использует жёсткие правила (это ансамбль из 200 деревьев), однако анализ структур деревьев позволяет выделить **де-факто пороги** для каждого класса в исходных (raw) единицах:

**Высокая усталость (High) — класс 0**

Наступает, когда выполняется **хотя бы 2** из следующих условий:

| № | Условие (raw) | Scaled порог | Доля деревьев, использующих этот порог |
|---|---------------|:------------:|:-------------------------------------:|
| 1 | Hours_Awake **> 14** ч | > 1.00 | ~65% деревьев |
| 2 | Decisions_Made **> 60** | > 0.57 | ~72% деревьев |
| 3 | Sleep_Hours_Last_Night **< 5** ч | < -0.97 | ~58% деревьев |
| 4 | Error_Rate **> 0.15** | > 2.05 | ~45% деревьев |
| 5 | Cognitive_Load_Score **> 0.6** | > — | ~40% деревьев |

**Типичный профиль High:** Hours_Awake=16, Decisions_Made=80, Error_Rate=0.25, Sleep=4ч → вероятность High ≈ **80.78%** (Low: 0.5%, Moderate: 18.72%)

---

**Умеренная усталость (Moderate) — класс 2**

| № | Условие (raw) | Scaled порог | Пояснение |
|---|---------------|:------------:|-----------|
| 1 | Hours_Awake **10-14** ч | 0.18 до 1.00 | Пограничное время бодрствования |
| 2 | Decisions_Made **30-60** | -0.60 до 0.57 | Умеренная рабочая нагрузка |
| 3 | Sleep_Hours_Last_Night **5-7** ч | -0.97 до 0.83 | Недостаточный, но не критический сон |
| 4 | Error_Rate **0.05-0.15** | 0.31 до 2.05 | Повышенная, но не критическая ошибка |

**Типичный профиль Moderate:** Hours_Awake=12, Decisions_Made=45, Error_Rate=0.10, Sleep=6ч → вероятности: High=51.19%, Low=6.30%, Moderate=42.50%

> **Важно:** При Moderate профиле модель часто даёт пограничный прогноз (High vs Moderate ~50/50). Это conservatism bias — модель склонна классифицировать пограничные случаи как более опасный класс.

---

**Низкая усталость (Low) — класс 1**

Выполняются **все** из следующих условий:

| № | Условие (raw) | Scaled порог |
|---|---------------|:------------:|
| 1 | Hours_Awake **< 10** ч | < 0.18 |
| 2 | Decisions_Made **< 30** | < -0.60 |
| 3 | Sleep_Hours_Last_Night **≥ 7** ч | ≥ 0.83 |
| 4 | Error_Rate **< 0.05** | < 0.31 |
| 5 | Caffeine_Intake_Cups **< 3** | < 0.77 |

**Типичный профиль Low:** Hours_Awake=6, Decisions_Made=15, Error_Rate=0.02, Sleep=8ч → вероятность Low = **52.50%** (High: 3.75%, Moderate: 43.75%)

> Почему Low confidence всего 52.5% для явно бодрствующего пилота? Из-за несбалансированности классов: Low — 47%, High — 33.6%, Moderate — 19.5%. Модель консервативна и резервирует 43.75% вероятности за Moderate, поскольку абсолютно «чистых» Low-сэмплов в обучающей выборке меньше.

## 2. CNN модель (Face Fatigue Detection)

### Архитектура: ResNet18 Transfer Learning

```python
# Базовая модель: ResNet18 предобученный на ImageNet
model = torchvision.models.resnet18(weights=ImageNet_Weights.DEFAULT)
model.fc = nn.Linear(512, 2)  # 2 класса: Fatigue (усталость), NonFatigue
```

### Гиперпараметры

| Параметр | Значение | Обоснование |
|----------|---------|-------------|
| Batch Size | 32 | Баланс скорости/памяти |
| Learning Rate | 0.001 | Стандарт для transfer learning |
| Optimizer | Adam | Быстрая сходимость |
| Image Size | 224x224 | Стандарт ResNet |
| Epochs | 10 | Ранняя остановка для избежания переобучения |

### Аугментация данных

```python
transforms.Compose([
    transforms.Resize((224, 224)),
    transforms.RandomHorizontalFlip(),     # Имитация разных ракурсов
    transforms.RandomRotation(10),        # Небольшие повороты головы
    transforms.ColorJitter(brightness=0.2, contrast=0.2),  # Разное освещение в кабине
    transforms.ToTensor(),
    transforms.Normalize(mean=[0.485, 0.456, 0.406], std=[0.229, 0.224, 0.225])
])
```

### Результаты обучения

| Epoch | Train Loss | Train Acc | Val Loss | Val Acc |
|-------|------------|-----------|----------|---------|
| 1 | 0.4307 | 82.61% | 0.2618 | **87.73%** |
| 2 | 0.1866 | 93.01% | 0.3277 | **88.64%** |

### Почему ResNet18?

| Критерий | ResNet18 | ResNet50 | EfficientNet | VGG16 |
|----------|----------|----------|--------------|-------|
| Размер | 11 MB | 25 MB | 5 MB | 138 MB |
| Скорость | Быстро | Медленно | Быстро | Медленно |
| Precision | 69.8% | 76.1% | 77.1% | 71.5% |
| Для 2200 фото | ✓ | Переобучение | ✓ | Переобучение |

**Обоснование выбора:**

1. **Малый датасет (2200 изображений)**
   - ResNet18 менее подвержен переобучению
   - Предобученные веса компенсируют малое количество данных

2. **Transfer Learning**
   - ImageNet признаки (края, текстуры) применимы к лицам
   - Достаточно 2 эпох fine-tuning

3. **Скорость inference**
   - Важно для real-time предполётного контроля
   - Работает на CPU без GPU

4. **Точность 88.64% приемлема**
   - Дополняет tabular анализ
   - Double-check для критичных случаев

### Потенциал улучшения CNN

1. **Больше данных** - 10K+ изображений для лучшей генерализации
2. **Face Detection** - предварительное обнаружение лица (MTCNN)
3. **Eye Aspect Ratio** - специализированные признаки глаз
4. **Yawning Detection** - обнаружение зевания
5. **Ensemble** - комбинация ResNet + EfficientNet

## 4. Рекомендации по уровням усталости

### Высокая усталость (High)

**Признаки:**
- Часы бодрствования > 14
- Принято решений > 60
- Сон < 5 часов
- Error_Rate > 0.15

**Действия:**
- &#x274C; НЕМЕДЛЕННО прекратить выполнение полётных обязанностей
- &#x1F6AB; Не допускать к выполнению рейса
- &#x1F4DE; Сообщить руководству
- &#x1F4F1; Обратиться за медицинской помощью

**Рекомендуемый отдых:** 12-14 часов

---

### Умеренная усталость (Moderate)

**Признаки:**
- Часы бодрствования 10-14
- Принято решений 30-60
- Сон 5-7 часов
- Error_Rate 0.05-0.15

**Действия:**
- &#x26A0; Рекомендуется отмена или перенос некритичных рейсов
- &#x1F4AC; Провести дополнительный брифинг с вторым пилотом
- &#x23F0; Увеличить частоту перерывов
- &#x1F4A4; Избегать сложных метеоусловий

**Рекомендуемый отдых:** 8-10 часов

---

### Низкая усталость (Low)

**Признаки:**
- Часы бодрствования < 10
- Принято решений < 30
- Сон > 7 часов
- Error_Rate < 0.05

**Действия:**
- &#x2705; Можно выполнять полётные обязанности
- &#x1F4CB; Следовать стандартным процедурам
- &#x23F0; Поддерживать регулярные перерывы

**Рекомендуемый отдых:** Обычный
